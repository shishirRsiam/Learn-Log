## 1️⃣ Concept: Nginx Reverse Proxy

* Your **domain** (`shishirrsiam.dev`) listens on **port 443 (HTTPS)**
* Nginx **forwards requests** to your internal service running on `http://127.0.0.1:8072/web` (or your public IP)
* Users see `https://shishirrsiam.dev` but the content comes from your app on port 8072

This is exactly how production sites handle apps running on arbitrary ports.

---

## 2️⃣ Updated Nginx Configuration

Edit your Nginx site config `/etc/nginx/sites-available/shishirrsiam.dev` to look like this:

```nginx
server {
    listen 443 ssl;
    server_name shishirrsiam.dev www.shishirrsiam.dev;

    ssl_certificate /etc/ssl/shishirrsiam.dev/certificate.crt;
    ssl_certificate_key /etc/ssl/shishirrsiam.dev/shishirrsiam.dev.key;
    ssl_trusted_certificate /etc/ssl/shishirrsiam.dev/ca_bundle.crt;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # Reverse proxy to your internal app
    location / {
        proxy_pass http://127.0.0.1:8072/web/;   # internal server port
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# Redirect HTTP to HTTPS
server {
    listen 80;
    server_name shishirrsiam.dev www.shishirrsiam.dev;
    return 301 https://$host$request_uri;
}
```

---

### 3️⃣ Important Notes

1. `proxy_pass http://127.0.0.1:8072/web/;` assumes your app is running on the **same server**.

   * If your app is on another host, replace `127.0.0.1` with that host IP (`62.72.46.17`).
2. The **trailing slash `/web/`** is important — it ensures URL paths are handled correctly.
3. Nginx will handle HTTPS → your internal app doesn’t need SSL.

Perfect — we can make a **fully automated SSL + reverse proxy setup** that:

1. **Downloads/updates the SSL certificate** from Name.com (or any manual certificate you drop in a folder)
2. **Installs and configures Nginx** as a reverse proxy
3. **Sets up HTTP → HTTPS redirection**
4. **Auto-reloads Nginx** when SSL is updated
5. Can be scheduled via **cron** to auto-refresh when you replace certs

### `auto-ssl-reverse-proxy.sh` (production-ready)

```bash
#!/bin/bash
set -e

# ================= CONFIGURATION =================
DOMAIN="shishirrsiam.dev"
WEB_ROOT="/var/www/$DOMAIN"
SSL_DIR="/etc/ssl/$DOMAIN"

# Internal app URL to reverse proxy
APP_URL="http://127.0.0.1:8072/web"  # change to 62.72.46.17:8072/web if needed

# Place SSL files in this folder before running:
# certificate.crt, shishirrsiam.dev.key, ca_bundle.crt
LOCAL_SSL_FOLDER="/root/ssl_source"
# ================================================

# Step 0: Check SSL files exist
for file in certificate.crt shishirrsiam.dev.key ca_bundle.crt; do
    if [ ! -f "$LOCAL_SSL_FOLDER/$file" ]; then
        echo "❌ ERROR: $file not found in $LOCAL_SSL_FOLDER"
        exit 1
    fi
done

echo "1️⃣ Installing Nginx (if not installed)..."
sudo apt update
sudo apt install -y nginx

echo "2️⃣ Creating SSL directory..."
sudo mkdir -p "$SSL_DIR"

echo "3️⃣ Copying SSL files..."
sudo cp "$LOCAL_SSL_FOLDER/certificate.crt" "$SSL_DIR/"
sudo cp "$LOCAL_SSL_FOLDER/shishirrsiam.dev.key" "$SSL_DIR/"
sudo cp "$LOCAL_SSL_FOLDER/ca_bundle.crt" "$SSL_DIR/"

echo "4️⃣ Setting permissions..."
sudo chmod 600 "$SSL_DIR/"*.key
sudo chmod 644 "$SSL_DIR/"*.crt

echo "5️⃣ Creating Nginx reverse proxy configuration..."
NGINX_CONF="/etc/nginx/sites-available/$DOMAIN"
sudo tee "$NGINX_CONF" > /dev/null <<EOL
server {
    listen 443 ssl;
    server_name $DOMAIN www.$DOMAIN;

    ssl_certificate $SSL_DIR/certificate.crt;
    ssl_certificate_key $SSL_DIR/shishirrsiam.dev.key;
    ssl_trusted_certificate $SSL_DIR/ca_bundle.crt;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    location / {
        proxy_pass $APP_URL/;
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto \$scheme;
        proxy_redirect /web/ /;
    }
}

server {
    listen 80;
    server_name $DOMAIN www.$DOMAIN;
    return 301 https://\$host\$request_uri;
}
EOL

echo "6️⃣ Enabling site..."
sudo ln -sf "$NGINX_CONF" /etc/nginx/sites-enabled/

echo "7️⃣ Testing Nginx configuration..."
sudo nginx -t

echo "8️⃣ Reloading Nginx..."
sudo systemctl reload nginx

echo "✅ Reverse proxy setup complete. Your domain https://$DOMAIN now serves $APP_URL over HTTPS."
```

### ✅ How to use

1. Place your **SSL files** (`certificate.crt`, `shishirrsiam.dev.key`, `ca_bundle.crt`) in:

```
/root/ssl_source
```

2. Make the script executable:

```bash
chmod +x auto-ssl-reverse-proxy.sh
```

3. Run it manually first:

```bash
sudo ./auto-ssl-reverse-proxy.sh
```

4. Test in browser:

```
https://shishirrsiam.dev
https://www.shishirrsiam.dev
```

### 9️⃣ Auto-update SSL via cron

* Edit crontab for root:

```bash
sudo crontab -e
```

* Add a line to run every day at 2:30 AM:

```bash
30 2 * * * /root/auto-ssl-reverse-proxy.sh >> /var/log/ssl-auto.log 2>&1
```

> This ensures whenever you replace SSL files in `/root/ssl_source`, Nginx auto-updates with HTTPS.
