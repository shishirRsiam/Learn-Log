## First: what a CSR actually is (so you don’t blindly copy-paste)

A **CSR (Certificate Signing Request)** is:

* A cryptographic request **generated on the server where SSL will be installed**
* It contains:

  * Your **domain** (`www.shishirrsiam.dev`)
  * Your **company/contact info**
  * A **public key**
* It is paired with a **private key** that must stay on that server

👉 **Key rule (non-negotiable):**
If you generate the CSR on *Server A*, the SSL **must be installed on Server A**.

## Correct Way: Generate CSR on Linux Server (Recommended)

### Step 1: SSH into your server

```bash
ssh root@your_server_ip
```

---

### Step 2: Create a directory for SSL (clean practice)

```bash
mkdir -p /etc/ssl/shishirrsiam.dev
cd /etc/ssl/shishirrsiam.dev
```

---

### Step 3: Generate **PRIVATE KEY + CSR**

Run **exactly this** (replace nothing except domain if needed):

```bash
openssl req -new -newkey rsa:2048 -nodes \
-keyout shishirrsiam.dev.key \
-out shishirrsiam.dev.csr
```

### Step 5: Copy the CSR content

```bash
cat shishirrsiam.dev.csr
```

You’ll see something like:

```

MIIC...
...
-----END CERTIFICATE REQUEST-----
```

👉 **Copy EVERYTHING**, including BEGIN and END lines.

---

### Step 6: Paste into Name.com CSR field

Paste that full block into:

> **Certificate Signing Request ***

Submit → continue SSL validation.

## ⚠️ Common Fatal Mistakes (avoid these)

* ❌ Generating CSR on your **local Windows machine**
* ❌ Using `shishirrsiam.dev` instead of **`www.shishirrsiam.dev`**
* ❌ Using Vercel/Netlify AND buying SSL
* ❌ Losing the `.key` file

👉 **Backup this file immediately:**

```bash
shishirrsiam.dev.key
```

If you lose it → SSL becomes useless.
